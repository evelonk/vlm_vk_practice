# VLM для русского VQA: обучение и оценка на GQA-ru / MMBench-ru

## Цель проекта

Дообучить несколько открытых vision-language моделей (VLM) отвечать на вопросы по
изображениям на русском языке и оценить эффект различных методов дообучения на
бенчмарках **GQA-ru** и **MMBench-ru**, опубликованных VK (deepvk). Отдельная часть цели - это
понять, насколько выбранные методы дообучения и используемые метрики адекватно отражают
реальное изменение качества модели, а не только формальное число

## Задачи

1. Изучить открытые датасеты deepvk для VLM: обучающий набор `LLaVA-Instruct-ru` и
   тестовые бенчмарки `GQA-ru`, `MMBench-ru`.
2. Выбрать модели и методы дообучения, реалистичные для одного GPU уровня T4.
3. Замерить baseline-качество базовых моделей на обоих бенчмарках без дообучения
4. Дообучить модели и сравнить два измерения: метод дообучения (LoRA против полного
   файнтюнинга декодера) и выбор базовой модели (Qwen2-VL-2B vs PaliGemma2-3B)
5. Проанализировать изменения метрик после дообучения — включая случаи, где строгая
   метрика ExactMatch не отражает реальное изменение качества ответа
6. Продемонстрировать захват пространственных свойств изображений через grounding —
   визуальную локализацию объектов (вывод bounding box) на модели Qwen2-VL, которая
   поддерживает эту способность из коробки, без дообучения
7. Оформить решение: код, веса, таблицу метрик, описание находок и ограничений

## Как использовались данные VK

| Датасет | Назначение в проекте |
|---|---|
| `deepvk/LLaVA-Instruct-ru` | Обучающие пары «изображение + вопрос/инструкция + ответ» на русском — источник для дообучения всех моделей. |
| `deepvk/GQA-ru` | Отложенный тест визуального рассуждения (короткий ответ), метрика ExactMatch. Использован только для оценки. |
| `deepvk/MMBench-ru` | Отложенный тест с вопросами с вариантами ответов (A/B/C/D), 3910 примеров. Использован только для оценки. |

Обучающие и тестовые данные не пересекаются.

## Модели и методы

Три независимых прогона:

1. **Qwen2-VL-2B-Instruct + LoRA** — компактная (2B) открытая мультиязычная VLM , дообученная методом LoRA (4-битное квантование весов + обучаемые
   адаптеры на проекциях внимания)
2. **Qwen2-VL-2B-Instruct + полный файнтюнинг декодера** — та же базовая модель, но вместо
   LoRA обучаются все веса языковой части при замороженном визуальном энкодере. Сравнение
   с (1) показывает, что дает полный файнтюнинг против легкого дообучения на одной и той же
   модели
3. **PaliGemma2-3B (mix) + LoRA** — модель другой архитектуры (SigLIP + Gemma 2), дообученная тем же методом LoRA, что и (1). Сравнение с (1) показывает разницу между базовыми моделями при одинаковом методе дообучения.

**Насколько мало весов реально обучается в LoRA-прогонах** — конкретные цифры из логов
обучения: у Qwen2-VL-2B дообучается **0.20%** параметров (4.36 млн из 2.21 млрд), у
PaliGemma2-3B — **0.31%** (9.38 млн из 3.04 млрд). Это прямое следствие ограничения ресурсов: один GPU уровня T4/P100(в гугл колаб или на кагле) с 16 ГБ памяти. Полный файнтюнинг всех весов потребовал бы в разы больше памяти под состояния оптимизатора, LoRA обучает на три порядка меньше
параметров, оставаясь единственным реалистичным вариантом.

## Ограничения

Обучение проведено на подвыборке из **400 примеров** `LLaVA-Instruct-ru`, а не на полном
датасете (**109 905** примеров), причем это не техническая недоработка, а из-за ограничения мощностей для обучения, так как лимиты постоянно заканчиваются. По измеренной скорости (Qwen2-VL-2B + LoRA, 6.3 сек на
пример) обучение на полном датасете заняло бы около **19 суток** непрерывного счета;
для сравнения, бесплатная квота Kaggle — порядка **30 GPU-часов в неделю** суммарно на все
задачи, а нужно было прогнать три модели, поэтому полученные результаты стоит
интерпретировать именно в этом контексте, а не как окончательную оценку потенциала методов
на большем объеме данных
